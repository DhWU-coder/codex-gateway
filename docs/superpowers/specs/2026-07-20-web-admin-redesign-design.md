# Codex Gateway Web 管理后台重构设计

## 背景

当前 Web UI 已具备服务状态、频道状态、实时会话、历史归档、AI 总结、连接测试和过程回复开关，但页面仍以调试信息展示为主。配置只显示文本摘要，用量日志和服务日志没有可视化入口，会话过程也缺少适合长期运维的详情界面。

本次重构参考 Claudish Web UI 的信息架构、视觉密度和频道管理交互，但不直接复制其 Provider、Web Chat、多语言等无关模块。实现继续使用 Bun 原生 HTTP 服务和浏览器原生 JavaScript，不引入 React、Vite 或新的前端构建链。

## 范围

管理后台包含以下顶层模块：

- 概览：服务运行状态、频道状态、活跃会话、最近请求和服务操作。
- 用量：Token 汇总、时间范围筛选、模型与工作目录分组、时间趋势和最近调用。
- 配置：展示项目内 `config.yaml`、Codex 运行参数和频道配置概览。
- 频道：管理飞书账号、测试连接、查看会话与历史归档。
- 日志：增量查看服务日志并支持搜索、级别筛选、暂停、复制和下载。

本次不实现 Web Chat。飞书仍是 Gateway 的聊天入口。

## 页面结构

页面顶部显示 Codex Gateway、运行状态、PID、主题切换和刷新操作。顶层导航采用 Claudish 风格的紧凑按钮标签。

### 概览

概览使用扫描友好的指标带和状态列表，不使用营销式大卡片。主要内容包括 PID、启动时间、端口、工作目录、配置路径、日志路径、频道连接数、活跃会话数和最近请求。停止和重启服务放在概览操作区，并对危险操作二次确认。

### 用量

用量页读取项目下 `.codex-usage/usage.jsonl`。页面支持今日、本周、本月、全部、最近和自定义日期范围，以及日、周、月时间桶。汇总指标包括请求数、总 Token、输入、缓存输入、输出和推理 Token。

趋势区展示按时间桶聚合的数据；分组区展示模型和工作目录；最近调用列表展示时间、模型、工作目录、请求 ID 和 Token 明细。空文件和不存在的文件显示明确空状态。

### 配置

配置页显示配置文件路径、服务参数、Codex 命令、模型、sandbox、profile、search 和附加参数。非热更新字段保持只读，并标注其当前有效值。

页面同时展示飞书频道账号概览，并提供跳转到频道页的入口。配置文件只允许通过专用结构化编辑器修改飞书账号的热更新字段，不提供原始 YAML 文本编辑器。

### 频道

频道页采用左右双栏：左侧展示频道类型及配置账号数、启用账号数、会话数和处理中数量；右侧展示飞书账号工作区。

每个飞书账号是一个独立重复项，显示账号 ID、启用状态、模型、工作目录、连接状态和会话数量。账号操作包括连接测试、编辑、删除；页面顶部支持添加账号和批量编辑。

可编辑字段为账号 ID、App ID、App Secret、Bot Open ID、域名、启用状态和实时过程回复。模型、工作目录、session 和历史策略只读，与 Claudish 的热更新边界保持一致。

账号下方使用紧凑行展示最近会话，包括来源、摘要、阶段、耗时和详情入口。点击后打开右侧大抽屉。

### 会话抽屉

抽屉按真实消息顺序展示用户消息、Codex 回复、工具调用、附件和错误。工具输入、工具结果、stderr 和原始事件默认折叠，长内容在固定区域内滚动。

抽屉提供实时过程和历史归档两个视图。历史归档根据当前会话的 `conversationKey` 自动加载，不要求用户手工输入。归档可查看完整消息并执行 AI 总结或刷新总结。

### 日志

日志页只读取当前服务状态中的 `service.log`，不接受客户端提供任意路径。服务端按最大字节数读取文件尾部，并使用游标支持增量刷新。

页面支持自动刷新、暂停滚动、文本搜索、日志级别筛选、复制当前结果和下载日志。文件不存在时返回空结果；文件被截断或轮转时游标自动重置。

## 视觉系统

视觉参考 Claudish：中性灰背景、清晰边框、青绿色强调色、紧凑排版和 6 至 8 像素圆角。错误使用红色，等待和处理中状态使用琥珀色，避免界面只依赖单一色相。

浅色和深色主题使用同一套语义变量。首次访问跟随系统主题，用户切换后写入 `localStorage`，并在 CSS 加载前设置主题以避免闪烁。

桌面端频道页使用 `280-360px + minmax(0, 1fr)` 双栏。窄屏切换为单栏，顶层导航横向滚动，会话抽屉占满视口。路径、错误、日志和回复文本必须正确截断、换行或滚动，不允许与操作控件重叠。

## 模块边界

现有 `src/web-server.ts` 保留 Web 服务启动、顶层请求分发和兼容导出。新增 `src/web/` 目录：

- `page.ts`：返回完整 HTML、CSS 和浏览器脚本。
- `usage-service.ts`：解析并聚合项目用量日志。
- `log-service.ts`：受限读取服务日志尾部。
- `config-editor.ts`：读取、脱敏、校验和保存飞书账号配置。
- `types.ts`：Web API 使用的共享类型。

`ChannelManager` 从固定数组改为按 channel ID 管理的 Map，并记录当前飞书账号配置。它负责新增、删除、原地更新和重建单个 Channel。

Daemon 监听实际传入的项目配置路径。文件变化后 500ms 防抖，成功解析后交给 ChannelManager 重载。解析或重载失败时保留当前运行中的频道，并记录最近一次重载错误。

## 配置热更新

配置编辑器使用 `yaml` 的 Document API 修改 `channels.feishu.accounts`，保留其他字段和注释。保存通过同目录临时文件加 rename 原子替换。

配置重载规则：

- 新账号：创建并在服务运行时启动对应 Channel。
- 删除账号：停止并移除对应 Channel。
- `enabled`、`appId`、`appSecret`、`botOpenId`、`domain` 变化：停止并重建对应 Channel。
- `sendProgressReplies` 变化：调用 `updateConfig` 原地更新。
- `model`、`cwd`、历史策略等非热字段：保留当前运行值，页面只读。

重建新 Channel 失败时，尽量恢复旧 Channel，并在重载状态和日志中记录错误，避免单个账号配置错误影响其他账号。

## API

保留现有 API，并新增：

- `GET /api/overview`：服务、配置路径、频道统计和最近重载状态。
- `GET /api/usage`：返回按查询参数聚合的用量数据。
- `GET /api/config`：返回脱敏后的公共配置状态。
- `GET /api/feishu-config`：返回飞书账号编辑状态。
- `GET /api/feishu-config/:id/secret`：按需返回本地账号 Secret。
- `POST /api/feishu-config`：校验并保存账号配置。
- `GET /api/logs`：返回受限日志尾部或游标后的增量内容。
- `GET /api/logs/download`：下载当前服务日志。
- `POST /api/service/restart`：安排后台辅助进程重启服务并返回 `202`。

所有 API 只在 `127.0.0.1` 暴露。未知频道、非法参数和配置校验错误返回 4xx；内部读取、保存和重载错误返回结构化 5xx。敏感字段不出现在普通状态、原始状态或日志 API 中。

## 前端状态与刷新

服务与频道状态每 2 秒刷新。前端使用请求序号或 AbortController 丢弃过期响应，并保持当前 tab、账号、会话和抽屉选择。

用量只在进入页面、修改筛选或手动刷新时请求。日志在页面激活且未暂停时增量刷新。归档和 AI 总结仅在打开对应会话后按需获取。

配置保存失败时保留表单草稿并在账号条目内显示错误。连接测试逐项显示检查结果和耗时。停止、重启和删除账号必须确认。

## 测试策略

- 配置编辑器测试覆盖 Secret 脱敏与按需读取、注释保留、原子写入、重复 ID、缺少凭据和非法域名。
- ChannelManager 测试覆盖新增、删除、原地更新、单账号重建、非热字段保留和重建失败恢复。
- Daemon 测试覆盖配置文件监听、防抖、成功重载和错误隔离。
- 用量服务测试覆盖日期范围、Token 聚合、模型与工作目录分组、时间桶和损坏 JSONL。
- 日志服务测试覆盖尾部读取、增量游标、文件轮转、不存在文件和路径边界。
- Web API 测试覆盖所有新路由、错误状态、Secret 泄漏防护和服务重启响应。
- 页面测试覆盖顶层模块、主题、账号编辑、会话抽屉、归档和日志控件。
- 使用 Playwright 在桌面和移动尺寸检查浅色、深色主题、无溢出、无重叠和抽屉交互。
- 最终运行 `bun test`、`bun run typecheck`、`bun run build` 和 `git diff --check`。

## 非目标

- 不实现 Web Chat 或浏览器终端。
- 不引入 Provider 管理、多语言或外部认证。
- 不允许浏览器读取任意本地文件。
- 不在本次重构中改变飞书消息协议、Codex CLI 调用方式或历史文件格式。
