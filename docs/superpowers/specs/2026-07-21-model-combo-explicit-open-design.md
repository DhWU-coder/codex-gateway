# 模型下拉显式展开设计

## 背景

管理台进入 Codex 配置编辑状态后会自动聚焦模型输入框。模型组合框当前把 `focus` 事件也作为展开下拉菜单的触发条件，因此点击“编辑配置”会立刻遮挡下方配置内容。

## 交互目标

- 点击“编辑配置”后保留模型输入框的自动焦点，方便直接键盘输入。
- 单纯获得焦点不展开模型列表。
- 用户点击输入框、输入内容、按上下方向键或点击右侧箭头时展开列表。
- 选择模型后下拉保持关闭，焦点返回输入框。
- 全局配置和飞书账号配置共用同一套模型组合框行为。

## 实现

保留 `editCodexModel` 中的输入框聚焦逻辑，删除 `createModelCombo` 对输入框 `focus` 事件的展开监听。现有 `click`、`input`、键盘和箭头按钮监听继续负责显式展开，不改变模型筛选、选择和保存逻辑。

## 验证

页面契约测试确认自动聚焦仍存在、点击展开仍存在，并确认不再注册聚焦展开监听。完成后运行 Web 测试、全量测试、类型检查和构建，再重启服务验证新页面已加载。
