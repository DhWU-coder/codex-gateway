# Codex Gateway Web 模型配置设计

## 背景

当前 Web 管理台会展示全局 `codex.model` 和飞书账号 `model`，但两个位置都不可编辑。飞书账号保存接口还会刻意保留旧模型并忽略页面输入，配置热更新也把模型归类为非热字段。

本次改动让两个模型字段都能在 Web UI 中配置，并参考 Claudish 的组合输入框：用户既可以输入任意模型 ID，也可以从 Codex CLI 当前账号实际可用的模型列表中选择。

## 配置语义

- 全局模型保存到项目根目录 `config.yaml` 的 `codex.model`。
- 全局模型留空时删除该 YAML 字段，由 Codex CLI 使用自身默认模型。
- 飞书账号模型保存到 `channels.feishu.accounts[].model`。
- 账号模型留空时删除账号字段，并继承当前全局模型。
- 用户可以保存不在下拉列表中的自定义模型 ID；最终可用性由 Codex CLI 在调用时判断。

## 模型目录

新增独立的 Codex 模型目录服务，通过配置中的 `codex.command` 启动 `codex app-server --stdio`。服务完成 `initialize` 握手后调用 `model/list`，读取模型 ID、显示名、说明和默认标记。

模型列表在服务进程内短时缓存，避免每次打开输入框都启动子进程。读取设置超时并确保子进程在成功、失败和超时后退出。读取失败时 API 返回明确错误，但页面仍保留自由输入能力。

Web API 新增 `GET /api/models`，只返回组合输入框需要的非敏感模型元数据。默认模型排在列表顶部，其余保持 CLI 顺序。

## Web 交互

配置页的 “Codex 配置” 标题栏增加编辑、保存和取消操作。非编辑状态继续使用紧凑定义列表；编辑状态只将模型一项替换为组合输入框，其他 Codex 参数保持只读。

频道页进入单账号或批量编辑后，账号模型字段从只读输入框变为组合输入框。账号字段为空时显示“继承全局模型”的占位提示，账号标题和状态区域继续显示当前有效模型。

组合输入框包括文本输入、下拉按钮和浮层列表，支持：

- 按模型 ID、显示名过滤。
- 鼠标选择。
- 上下方向键移动、Enter 确认、Escape 关闭。
- 输入列表外的模型 ID。
- CLI 默认模型标记和模型说明。
- 点击控件外部关闭。

## 配置保存

配置编辑器继续使用 `yaml` Document API 修改指定字段，保留其他配置、顺序和注释。写入仍使用同目录临时文件、`0600` 权限和原子 rename。

新增全局 Codex 模型编辑状态与保存函数，并扩展飞书账号保存逻辑：输入中明确提供 `model` 时写入或删除模型字段；未提供时保留旧值，兼容既有 API 调用。

## 无中断热更新

模型变化不重启服务，也不重连飞书频道。ChannelManager 将模型作为可热更新字段传给对应 Channel，Channel 再更新 SessionRouter 的默认模型。

会话模型在创建时写入 session 元数据。模型配置变化后：

- 正在运行和已有会话继续使用创建时记录的模型。
- `/new`、新私聊或新群聊会话使用新的有效模型。
- 账号显式模型优先于全局模型。
- AI 总结没有显式 `summary.model` 时使用更新后的账号默认模型。

为保证上述语义，Runner 输入优先使用 session 元数据中的模型，而不是每次读取当前默认值。

## 错误处理

- `model/list` 启动失败、协议错误或超时：`GET /api/models` 返回结构化错误，页面提示下拉不可用但不禁用输入。
- 全局模型保存失败：保留当前编辑值并显示错误。
- 账号模型保存失败：沿用现有账号编辑错误流程，不清空表单。
- 配置热更新失败：保留当前运行配置，并在概览的重载状态中展示错误。

## 测试

- 模型目录测试覆盖握手、列表解析、默认模型排序、缓存、子进程错误和超时。
- 配置编辑器测试覆盖全局模型写入/删除、注释保留、账号模型写入/删除/兼容保留。
- ChannelManager、FeishuChannel 和 SessionRouter 测试覆盖模型热更新、新旧会话模型边界和摘要模型回退。
- Web API 测试覆盖模型列表、全局模型读取与保存、错误状态。
- 页面测试覆盖两个组合输入框、编辑操作和账号继承提示。
- 最终运行 `bun test`、`bun run typecheck`、`bun run build` 和 `git diff --check`，再用 Playwright 检查桌面与移动端的浅色和深色布局。

## 非目标

- 不在 Gateway 中维护固定模型清单。
- 不配置 reasoning effort、service tier 或 personality。
- 不改变 Codex CLI 登录方式。
- 不自动迁移用户已有模型 ID。
